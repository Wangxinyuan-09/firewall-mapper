## 把流向树改成横向流动

把现在的"竖向树"调成从左到右的横向流动图，更贴近实际数据走向。

### 布局结构

```text
┌─────────┐      ┌─ #3  203.0.113.7:443  → :8443 ─┐      ┌──────────┐      ┌ permit tcp/8443 ┐
│         │      │ #4  203.0.113.7:80   → :8080  │      │          │      │ permit tcp/8080 │
│   src   │──┬──►│ #5  203.0.113.8:22   → :22    │──┬──►│   dst    │──┬──►│ permit tcp/22   │
│         │      └─ (DNAT 分支，竖排堆叠) ────────┘      │ [WAF]    │      │ deny  tcp/3306  │
└─────────┘                                              └──────────┘      └─────────────────┘
   源节点          DNAT 列 (多条竖向堆叠+左右连接线)        目的节点         策略端口 pill 列
```

横向 4 段：**源节点 → DNAT 列 → 目的节点 → 策略 pill 列**。多条 DNAT 竖向堆叠，左右各一段水平连接线，端点合拢到源/目的节点。直连时跳过 DNAT 段，源直接连到目的。

### 视觉细节

- 节点：圆角矩形大色块，源绿色边、目的蓝色边（带分类小标签如 WAF/网关）。
- DNAT 分支：每条 `#id  entryAddr:port → :backendPort` 一行小卡片，端口变化用 amber 高亮，命中 gap 的卡片整行带 amber 描边。
- 连接线：纯 CSS 横线（`h-px bg-border`）+ 竖向分支汇聚线，使用 flex + 绝对定位，无 SVG。
- 策略 pill：permit 绿、deny 红、其他灰，竖向堆叠在最右列。
- 视窗较窄时整段 `overflow-x-auto` 横向滚动，节点和分支保持原比例不换行。

### 不变的部分

- 卡片头部（src → DNAT×N → dst + 覆盖徽章）
- 卡片下方左右两栏 DNAT 明细 / 策略明细列表
- 顶部三联动筛选 + chip + literal 提示
- `src/lib/access.ts` 不动

### 文件改动

- `src/routes/access-graph.tsx`：替换 FlowCard 中部 "visual vertical tree" 这一节为新的横向 flow 段，新增 `HFanout` 横向分支组件、调整 `VLine` 为 `HLine`、`DnatNode` 改成横排紧凑卡片；其余结构保留。

### 不做

- 不引入图形库、不用 SVG / Canvas
- 不改解析器和数据层
- 不改 DNAT/策略明细列表的呈现
